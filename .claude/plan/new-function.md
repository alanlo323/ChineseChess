  建議實作順序

  第一批（低難度高回報）：
    Aspiration Window → Lazy Evaluation → IIR → Soft/Hard 時限

  第二批（中難度）：
    SEE → Pondering → ProbCut → Multi-PV

  長期目標：
    殘局知識強化 → NNUE

  最值得優先做的是 Aspiration Window：程式碼改動少（約 20 行加在迭代加深迴圈外層），但對中深度搜尋的節點削減效果立竿見影。
